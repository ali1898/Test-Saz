# Feature: Add `--scenario` and `--scenario-file` to `qa analyze`

## Goal
Allow users to pass a scenario file to `qa analyze` so that after login + page analysis, the tool uses the scenario to generate **focused** locators, page methods, and test steps — instead of generating artifacts for ALL detected elements.

## Current Behavior
- `qa analyze` extracts ALL interactive elements from the page
- Generates locators/page/test for every button, input, select, etc.
- Produces bloated artifacts with unused elements

## Desired Behavior
- User writes a scenario in Farsi/English (e.g., `scenarios/addMember.md`)
- Runs: `qa analyze -u "..." --scenario-file scenarios/addMember.md`
- After login + page analysis:
  1. Reads the scenario file
  2. Sends scenario + page analysis to LLM
  3. LLM generates **only** the locators, page methods, and test steps needed for the scenario
- Result: focused, clean artifacts

## Files to Modify

### 1. `packages/cli/src/index.ts` — Add CLI options
Add `--scenario` and `--scenario-file` options to the `analyze` command registration (~line 213):
```
.option("--scenario <text>", "inline scenario text (skips LLM scenario generation)")
.option("--scenario-file <path>", "read scenario from file (e.g., 'scenarios/addMember.md')")
```
Pass them to `analyzeCommand()`.

### 2. `packages/cli/src/commands/analyze.ts` — Handle scenario options
- Add `scenario?: string` and `scenarioFile?: string` to `AnalyzeOptions` interface
- In interactive mode: prompt "Do you have a scenario file? (y/n)" → if yes, ask for path
- If `scenarioFile` is provided, read the file content with `readFileSync`
- If `scenario` is provided (inline), use it directly
- Pass scenario content to `analyzeAndGenerate()`

### 3. `packages/core/src/generator/page-analyzer.ts` — Use scenario for LLM generation
- Add `scenario?: string` to `analyzeAndGenerate` options
- When scenario is provided:
  a. After `analyzePage()` extracts elements from the browser
  b. Build a combined prompt: scenario + extracted elements summary
  c. Call LLM to generate focused locators, page object, and test spec
  d. Write the LLM-generated artifacts (replacing the template-based generation)
- When no scenario: keep current behavior (template-based generation from all elements)

#### New function: `generateFromScenario()`
```typescript
async function generateFromScenario(
  provider: LLMProvider,
  scenario: string,
  analysis: PageAnalysis,
  pageName: string,
  options: { ... }
): Promise<{ locContent: string; pageContent: string; testContent: string }>
```

This function:
1. Builds a prompt with the scenario + element summary from `analysis`
2. Asks LLM to generate 3 files in sequence (locators → page → test)
3. Each prompt includes the scenario context and previous artifact outputs
4. Returns the 3 file contents

#### LLM Prompt Structure
```
System: QA automation expert, Cypress POM conventions

User (Phase 1 - Locators):
"You are generating locators for a Cypress test.
Scenario:
{scenario}

Page elements found:
{analysis.buttons, analysis.inputs, analysis.selects, etc.}

Generate ONLY the locators needed for the scenario steps.
Use single quotes for values: '[data-cy="..."]'
Flat structure, UPPER_SNAKE_CASE keys, JSDoc comments."

User (Phase 2 - Page Object):
"Generate a Page Object class for this scenario.
Import locators from: {importPath}
Methods should match scenario steps.
Return Cypress.Chainable<JQuery<HTMLElement>>."

User (Phase 3 - Test Spec):
"Generate a Cypress test spec implementing this scenario.
Import page from: {importPath}
Use describe/it blocks.
Call page methods for each scenario step."
```

### 4. `packages/core/src/generator/page-analyzer.ts` — Update exports
Add `generateFromScenario` to exports if needed.

## Example Usage
```bash
# Write scenario in Farsi
cat > scenarios/addMember.md << 'EOF'
# سناریو: افزودن عضو جدید

1. **ورود** به صفحه افزودن عضو
2. **انتخاب** نوع سند هویتی از لیست کشویی
3. **وارد کردن** شماره سند در فیلد مربوطه
4. **کلیک** روی دکمه استعلام
5. **بررسی** نمایش اطلاعات هویتی
6. **وارد کردن** اطلاعات تکمیلی
7. **کلیک** روی دکمه ذخیره
8. **بررسی** نمایش پیام موفقیت
EOF

# Run analyze with scenario
qa analyze \
  -u "http://testsiam.sk.local/Events/AddNewMember" \
  --login-url "http://testsiam.sk.local/login" \
  --username "siam.office.1309" \
  --password "1234qwer" \
  --scenario-file scenarios/addMember.md \
  --name "AddNewMember" \
  --debug -y \
  -p ./my-cypress-tests
```
